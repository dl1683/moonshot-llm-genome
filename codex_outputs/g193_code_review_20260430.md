**Findings**
1. **SEV-8: Resume can mix incompatible compiler rows across cells.**  
   g193 retrains/regenerates the compiler rows before loading an existing payload, then skips completed cells from the old payload. A resumed run can combine old cell results with newly generated rows and stale `compiler_stats`. See [compiler generation](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:432>) and [resume skip](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:458>). This invalidates incremental save correctness unless the compiler artifact/config/hash is persisted and checked.

2. **SEV-6: Compiler “holdout” is also the early-stopping validation set.**  
   The code selects `best_state` using holdout MSE, then reports that same holdout MSE/cosine as compiler quality. See [holdout selection](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:160>) and [final holdout reporting](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:179>). No target rows leak into compiler training, but the quality metric is validation-tuned, not an unbiased held-out test.

3. **SEV-6: Non-finite guards miss eval/generation paths.**  
   Shell train loss is guarded, but `_eval_nll` can return NaN/inf without a local guard, and JSON save only catches it after a full cell. Compiler holdout loss/cosine and generated rows are also unchecked. See [shell guard](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:261>), [g188 eval accumulation](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_188_tokenizer_flow_bridge.py:461>), and [JSON save](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:487>).

4. **SEV-6: Shuffled control preserves global row distribution, not token-conditioned stats.**  
   `compiled_shuffled = compiled_embed[perm]` destroys row-token association and preserves global Fro/distribution, but it does not preserve per-token row norm/frequency/length correlations. Since log-frequency is an input feature, a compiled-vs-shuffled gain can partly reflect token-conditioned scalar statistics, not only vector “content.” See [features](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:78>) and [shuffle](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:443>).

5. **SEV-6: Smoke/compiler epoch config is ignored.**  
   `compiler_epochs = 5 if smoke else COMPILER_EPOCHS` is recorded, but `train_compiler()` always uses global `COMPILER_EPOCHS=200`. Smoke runs can be much slower than intended and report false config. See [smoke config](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:361>) and [compiler loop](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_193_token_row_compiler.py:137>).
