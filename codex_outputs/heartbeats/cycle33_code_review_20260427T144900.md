- `SEV 8` [genome_165_annealed_donor.py](</C:/Users/devan/OneDrive/Desktop/Projects/AI Moonshots/moonshot-llm-genome/code/genome_165_annealed_donor.py:160>) `anchor_loss()` is the new hot path, but the waste is not just the Python `for` loop. Each anchored step builds a full FP32 autograd graph over all matched params and materializes `p.float() - donor` temporaries across the whole model. That is the best explanation for the jump from the prereg ~0.4s/step to the observed ~1.0s/step. A pure “vectorize the loop” rewrite probably will not cut 50%; the stronger fix is to remove anchor-loss autograd entirely and inject the exact gradient after CE backward:
```python
anchor_pairs = build_anchor_pairs(recipient, donor_params, submanifold)

optimizer.zero_grad(set_to_none=True)
loss_ce.backward()
if donor_params is not None and lam_t > 0:
    with torch.no_grad():
        for p, d in anchor_pairs:
            p.grad.add_((p.detach().float() - d).to(p.grad.dtype), alpha=2.0 * lam_t)
torch.nn.utils.clip_grad_norm_(recipient.parameters(), 1.0)
optimizer.step()
```
Delete the `loss_anchor = anchor_loss(...)` / `loss = loss_ce + lam_t * loss_anchor` path.
