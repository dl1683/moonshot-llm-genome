I’d approve g182 only with amendments.

1. **GPT-2 arch is genuinely distinct.** MHA + GELU + learned absolute positions + LayerNorm + GPT-2 BPE is enough for a reviewer to accept cross-architecture, provided it is a real `GPT2LMHeadModel`, not Qwen with a GPT-2 tokenizer. Caveat: param count differs: GPT-2 config is ~83M total / 44M non-embedding params; Qwen3-shell is ~169M / 52M. Call it matched width/depth/block-compute, not matched total params.

2. **10 seeds per arm is enough for phase 1.** Do not go above 10 before adding a third architecture. More architecture families buy more credibility than 15-20 seeds on two families. Use fixed seeds across arch: `42,7,13,101,211,307,401,503,601,709`.

3. **3600 steps / step 108 is the right primary horizon** because it matches existing g173/g180 infrastructure. But gate this on g181b: if 5000-step ranking changes materially, switch all g182 cells to `5000` with features at `150`. Do not mix 3600 and 5000 rows.

4. **Baselines are not comprehensive yet.** Add these locked baselines:
   - arm mean / protocol mean baseline
   - full telemetry baseline: loss trajectory + gradient stats + arm labels + arch/tokenizer/vocab/param covariates
   - learning-curve extrapolation from steps `10..108`
   - delayed-loss secondary baselines at `200` and `500`
   - within-arm residual baseline: subtract arm mean, then ask whether geometry predicts residuals

5. **Leave-one-arch-out is right, but weak with only 2 arch families.** Treat it as two transfer tests, not real architecture-level CV. Bootstrap by `(architecture, seed)` blocks containing all 3 arms. If you add Llama, then LOAO becomes much stronger.

6. **Compute is compliant only as a resumable sweep.** `60 x 21m = 21h`; the draft’s “5 sessions” is too tight. Use **6 sessions x 10 cells**, capped at `3.75h`. Default to one GPU training process. Allow 2 concurrent cells only after a two-cell smoke test proves `>=1.35x` throughput and peak VRAM `<18GB`. Checkpoint after every cell, cache features at step 108/150, atomic-write the result JSON.

7. **Biggest threat:** the predictor learns Qwen-reference/protocol fingerprints, not architecture-neutral training health. Preempt it by making a **reference-free geometry model co-primary**: drop `hidden_to_qwen_ref_*`, `embed_to_qwen_ref_*`, `lm_head_to_qwen_ref_*`, and `*_reference_rows_used`. Also do not use the heterogeneous 113-row g180 historical train set as primary g182 data; run a clean balanced 60-cell arena.

8. **Score:** current draft PASS is capped around **8.0/10**. With the amendments above, PASS is a real **9.0/10**. WEAK PASS is **6.5/10**, not 7+, unless a third architecture later confirms the surviving fold.
